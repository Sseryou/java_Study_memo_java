저장 후 스택 영역 메모리의 객체에 그 값이 담겨있는 힙 메모리의 주소값을 저장해 준다.




생성자
- 객체를 생성해 주는 함수
- 클래스의 정의 -> 필요한 자원 -> 메모리
- 생성자 함수의 호출 결과로써 객체에 접근할 수 있는 주소값을 반환

- 클래스명과 동일한 함수명
- 기본 기능 : 객체 생성기능 -> 숨겨져 있다
- 소스에서 통제 불가



1. 디폴트 생성자
	- 클래스 안에 생성자 메서드가 정이되어 있지 않은 경우
	- 컴파일러가 자동 생성해주는 생성자 메서드
	public 클래스명() {}

- 객체를 만드는 역할, 반환값 주소값
- 디폴트 생성자 안에는 객체 '생성 이후' 에 작업할 부분만 정의할 수 있음.
- 주로 하는 작업 : 생성된 객체의 멤버 변수 초기화
- 반환값은 주소값으로 고정이므로 반환값을 새로 정의할 수 없다.

- 매개변수를 넣어 객체 생성시에 기본값을 바로 정의하게 할 수도 있다.
- 디폴트 생성자도 생성자 오버로드가 적용되어, 값을 안넣으면 기본값, 값을 넣어서 기본값을 만들게 할 수도 있다.
- 생성자가 아무것도 없을때만 자동 생성

정보은닉

1. 접근제어자
	public : 접근 범위가 클래스, 내부 외부, 외부 패키지까지 접근
	protected : 동일 패키지에서만 클래스 내부, 외부 접근 가능(default)
			-외부 패키지에서는 상속을 톤해서 클래스 내부(private)에서 접근 가능
	default : 접근 제어자를 정의 안한 경우
		동일 패키지에서만 클래스 내부 외부에서 접근 가능
	private : 클래스 내부에서만 접근이 가능

2. 멤버 변수 -> private : 
- 멤버 변수는 값을 대입할때 값이 설정
- 멤버 변수는 값의 통제가 불가
- 접근 못하게 private

3. 멤버 변수의 값을 지정하고, 조회할 수 있는 메서드 추가
